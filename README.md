# RVT-Swarm

Prototype research codebase for **RVT-Swarm: Recoverability-Verified Topological Control for Decentralized Swarm Formation**.

This repo contains:
- the proposed RVT-Swarm method
- strong historical-style baselines implemented in a common simulator
- train / evaluate / ablation entrypoints
- a lightweight multi-robot formation environment for rapid iteration

## What is implemented

Implemented and runnable:
- lightweight 2D swarm environment with static and dynamic obstacles
- graph-based policy backbone in plain PyTorch
- proposed RVT-Swarm controller with:
  - control head
  - recoverability score-map head over candidate topologies
  - topology prior head
  - counterfactual topology selector with hysteresis
  - progress-preserving recoverability shield
- baselines:
  - `gnn_only`
  - `instant_cert`
  - `adaptive_formation`
  - `cbf_qp` using explicit CBF constraints and a per-robot QP solve
  - `orca` backed by the RVO2 ORCA library
  - `centralized_mpc` using finite-horizon centralized predictive search in the shared simulator model
  - `cbf_qp_like`
  - `orca_like`
- data generation from a heuristic expert and short-horizon recoverability rollouts
- training pipeline
- evaluation / comparison / ablations
- extra liveness metrics: stall rate, deadlock, topology switches
- visualization outputs for both episode GIFs and per-step metrics plots

Research approximations in this prototype:
- `cbf_qp_like` and `orca_like` are simplified research baselines, not exact reproductions of published packages.
- `orca` uses the Apache-licensed RVO2 ORCA implementation vendored under [third_party/Python-RVO2](/Users/udy/rvt/third_party/Python-RVO2).
- `cbf_qp` is an actual control-barrier-function safety filter around the fixed-formation nominal controller; it is an in-repo implementation rather than a wrapped external package.
- `centralized_mpc` is an actual receding-horizon controller in this repo, implemented as beam search over topology-conditioned nominal actions under the exact simulator dynamics.
- recoverability labels are generated by short-horizon counterfactual rollouts under candidate topology-control actions.
- topology actions are `keep`, `compress`, `line`, `split_hint`, `recover`.

## Quick start

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python run_experiments.py --mode train_all --device auto
python run_experiments.py --mode eval_all --device auto
python run_experiments.py --mode ablations --device auto
python run_experiments.py --mode visualize --device auto
```

The real `orca` baseline compiles the vendored RVO2 extension, so the install
step above requires a working C++ toolchain and CMake.

## Notes

The goal here is to give you a **paper-ready research scaffold** rather than a final benchmark-winning implementation. Compared with the initial scaffold, this version adds a more faithful counterfactual RVT pipeline by predicting per-topology recoverability scores and selecting/swapping topology online with hysteresis.


## Device selection

The repo supports `--device cpu`, `--device cuda`, `--device mps`, and `--device auto`.

- `auto` picks CUDA if available, otherwise Apple MPS on macOS, otherwise CPU.
- On Apple Silicon / macOS with Metal support, use `--device mps` or just `--device auto`.

Examples:

```bash
python run_experiments.py --mode train_all --device auto --results-dir results_auto
python run_experiments.py --mode eval_all --device mps --results-dir results_mps
```

You can verify MPS support with:

```bash
python -c "import torch; print('MPS built:', torch.backends.mps.is_built()); print('MPS available:', torch.backends.mps.is_available())"
```


## Training notes

- `gnn_only`: up to 120 epochs with patience-based early stopping
- `instant_cert`: up to 120 epochs with patience-based early stopping
- `rvt_swarm`: up to 300 epochs, with a 50-epoch auxiliary warmup and patience-based early stopping afterwards
- Best checkpoints are saved as `*_best.pt`; last checkpoints as `*_last.pt`.
